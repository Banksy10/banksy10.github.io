项目流程

Publish date: 2021-05-30
Tags: markdown, typography

RayTracing

1.搭建环境

  1. 下载：optix6.5.0 cuda10.1 cmake opencv3.20 ；bullet
  2. 安装cuda；optix在默认安装路径；安装opencv，cmake，bullet在指定的路径即可
  3. 在项目管理属性页修改其环境

注意：在修改链接器的附加依赖项时，一般下载的D:\feng\ray\WangQVer2\bullet3-master\bin\LinearMath_vs2010_x64_debug.lib没有此文件，此时应该在bullet3-master\build文件夹下面的LinearMath.slh生成该文件，再修改

4.在optix中的example中修改模型，模型的描述文件，注意optix的版本

注意：此处的描述文件可有人体器官的参数

在此处修改文件，上面的代码不用管
"meshes": [
    {
      "file": "onlyRoom.obj",
      "rigid": true,
      "vascular": false,
      "deltas": [0.0,0.0,0.0],
      "material": "MUSCULE",
      "outsideMaterial": "BLOOD",
      "outsideNormals": true
    }
  ],
  "origin": [0.0, 0.0, 0.0],
  "spacing": [1.0,1.0,1.0],
  "scaling": 0.1,
  "startingMaterial": "FAT"
}

5.生成图像

2.生成文件dll（光线追踪的调用文件）

（此处具体生成方法可依据网上所给的案例生成）

运行修改后的代码后，应该会在该项目文件夹中的x64下的debug下面的文件夹下面出现一系列文件以及dll文件，

3.编写一个c#调用脚本

在此处为c#调用c++程序，此处即可修改之前所生成的dll文件是否出现问题，若无问题可进行下一步，此处所生成的.cs为后面unity中的脚本，即实现在unity中对c++程序的调用

4.新建unity 3d文件

将模型拖入其项目中；将上一步中所产生的dll文件以及其他一系列文件放入Assets\Plugins

操作模型的源代码在asset\scripts中可查看

后续可对探头编写一个位置移动脚本

后续学习：

熟悉c#调用c++程序，cuda程序等；

熟悉unity软件的操作以及与HoloLens的结合；

新建一些模型的，同时也修改其描述文件；

了解关于optix 引擎的一些原理，以及之后更新的7.0版本：主要是做动态模型的模拟
